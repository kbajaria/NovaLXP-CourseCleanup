# Repos, README-driven work, and staying inside the workstation (15 minutes)

## What matters for this hackathon
You may still receive code from a source repository, but the key beginner skill here is not mastering a local Git workflow. The key skill is opening the correct project inside **Cloud Workstations** and following the README in order.

## Terms you must recognize
- **Repository (repo)**: the project folder and its history
- **Branch**: a line of work in the repository
- **Commit**: a saved change
- **README**: the instructions for setup, environment values, and how to run the app

For the hackathon, beginners mainly need to:
- open the repo
- read the README
- copy commands from the instructions
- stay in the correct workstation session while following the steps

## How to use a README well
Look for these sections:
- prerequisites
- setup
- environment variables
- run the app
- troubleshooting

## A safe beginner pattern
1. Read prerequisites first.
2. Do not skip install steps.
3. Run commands in order.
4. Compare your screen with the expected screenshots or output.
5. If the README assumes Cloud Workstations, do not substitute a local-machine workflow.
