# Agent bootstrapping (18-20 minutes)

## What "bootstrapping" means here
Bootstrapping means getting the first working version of the agent running.

Typical steps:
1. Clone or open the repo.
2. Install dependencies.
3. Add environment values.
4. Start the app.
5. Confirm the starter agent responds.

## Beginner checkpoints after each step
- After install: no fatal dependency errors
- After env setup: required keys are present
- After start: the app runs without crashing
- After first test: the agent gives some response, even if basic

## Common failure points
- forgot to install dependencies
- edited the wrong file
- started the app from the wrong folder
- pasted placeholder credentials instead of real ones
- used the wrong account or cloud project

## What success looks like
A minimal success is not "finished product." It is:
- the starter agent runs
- you can send an input
- the project is ready for the next tile's changes
