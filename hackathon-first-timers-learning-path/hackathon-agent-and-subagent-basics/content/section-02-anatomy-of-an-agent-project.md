# Anatomy of an agent project (12-15 minutes)

Many starter agent projects include some version of these parts:

- `README.md` -> how to set up and run the project
- source files -> where the agent logic lives
- config or env files -> API keys, model names, project IDs
- package file -> lists dependencies
- prompts or instruction files -> define the agent behavior

## A healthy beginner habit
Before changing anything:
1. Read the README.
2. Identify where environment values are expected.
3. Find the file that defines the main agent behavior.
4. Run the unmodified project once if possible.

## Why "run first, then change" matters
If the starter app already works before you edit it, later failures are easier to isolate.

## What to look for in code without reading everything
- function names
- comments
- file names like `agent`, `tools`, `config`, `prompts`, or `server`
- the place where the app starts
