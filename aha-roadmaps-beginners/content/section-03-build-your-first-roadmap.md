# Build your first roadmap in Aha! (20–25 minutes)

Goal: Create 1 release and 3–5 features, then visualize them on a features roadmap.

## Step 1 — Go to Releases and create a release (5 minutes)
1) Open Aha! Roadmaps and choose the correct workspace/product.
2) Navigate to **Releases**.
3) Click **Add release**.
4) Name it something like: “Q2 2026” (or your next quarter).
5) Set start/end dates (keep it simple).

### Visual reference (Aha! public knowledge base)
**Create a release**
https://support.aha.io/aha-roadmaps/support-articles/releases/create-release~7444673597282004033

## Step 2 — Add features to the release (10 minutes)
Create 3–5 features that a roadmap audience would understand.
Example set:
- Improve onboarding flow
- SSO enhancements
- Billing history export
- Admin audit trail

Two common ways:
- Add features from the release planning view
- Add features directly where they belong (via +)

### Visual reference (Aha! public knowledge base)
**Add features, phases, and milestones to a release**
https://support.aha.io/aha-roadmaps/support-articles/releases/add-features-phases-and-milestones-to-release~7444673630691952796

## Step 3 — View the features roadmap (5–8 minutes)
1) Navigate to **Roadmaps → Features**.
2) Confirm your new release and features appear.
3) Adjust the time scale / zoom so the quarter fits nicely.

### Visual reference (Aha! public knowledge base)

## Step 4 — Make it readable (2 minutes)
Before moving on:
- Hide anything your audience won’t understand
- Keep the view “clean” (just the fields needed)
